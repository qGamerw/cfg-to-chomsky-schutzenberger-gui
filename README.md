# Конвертер КСГ -> представление Хомского–Шютценбергера (Tkinter)

Приложение на Python/Tkinter для работы с контекстно-свободными грамматиками (КСГ):

1. парсит и валидирует грамматику,
2. строит представление Хомского–Шютценбергера: `L = h(R ∩ Dyck_Γ)`,
3. строит дерево разбора для введенной строки,
4. показывает краткий и подробный режимы вывода,
5. ведет логи действий и ошибок.

---

## Основные возможности

- Поддержка правил `A -> ... | ...` и `A → ...`.
- Поддержка пустой продукции: `ε`, `epsilon`, `""`.
- Поддержка продолжения альтернатив с новой строки через `|`.
- Корректная токенизация лексем (включая компактные формы вроде `P(NumProd)`).
- Литералы в кавычках обрабатываются как один терминал: `"read_write" -> read_write`.
- Построение:
  - `K = N ∪ Σ ∪ {⊥}`
  - `Γ = {[X, ]X | X ∈ K}`
  - `R` (через `STEP`)
  - `h`
- Графическое дерево разбора (узлы-кружки + связи) с масштабированием.
- Сохранение дерева разбора в PNG (сохраняется все дерево, не только видимая область).
- Загрузка/сохранение КСГ в `*.txt`.

## Вкладки интерфейса

### `Ввод грамматики`
- Поле ввода КСГ.
- Кнопки:
  - `Конвертировать`
  - `Построить дерево`
  - `Загрузить КСГ` (`.txt`)
  - `Сохранить КСГ` (`.txt`)
  - `Пример 1..4`

### `Вывод`
- Краткий понятный результат:
  - итоговые метрики,
  - обзор алфавитов,
  - где появляется вывод,
  - компактные формулы.

### `Подробный вывод`
- Полный отчет по алгоритму:
  - шаги построения,
  - секции `K`, `Γ`, `Dyck_Γ`, `R`,
  - детальная расшифровка `STEP`,
  - полный `h`.

### `Построить дерево`
- Ввод строки и рендер дерева разбора на `Canvas`.
- Кнопки `Масштаб +` / `Масштаб -`.
- Кнопка `Сохранить результат` (PNG).
- Горизонтальная/вертикальная прокрутка оставлена только в этой вкладке.

### `Описание теоремы`
- Текстовое пояснение теоремы Хомского–Шютценбергера.

### `Примеры`
- Быстрые кнопки загрузки примеров и `Конвертировать`.
- Кнопка `Демо`: пошаговый сценарий по всем вкладкам с модальными окнами пояснений.

### `Настройки`
- Изменение размера шрифта полей ввода/вывода.

### `Логи`
- Логи действий UI, шагов конвертации, ошибок.
- Логи в консоль отправляются через очередь и фоновый поток (UI не блокируется).

## Формат грамматики

Базовый пример:

```text
S -> a S b | A
A -> a A | ε
```

Лексемный пример:

```text
Scope -> read | write | read_write | admin
Expr -> P(NumProd) / P(DenProd)
```

Пример с кавычками:

```text
Scope -> "read" | "read_write" | "admin"
```

---

## Кэширование вычислений

- Конвертация (`K, Γ, R, h`) пересчитывается только если **изменился текст грамматики**.
- При повторном открытии вкладок `Вывод` / `Подробный вывод` без изменений используется кэш.
- Кэш дерева разбора учитывает пару `(текст грамматики, входная строка)`.

---

## Ввод строки для дерева

- Поддерживается ввод **с пробелами и без пробелов**.
- Для компактного ввода выполняется сегментация по терминалам грамматики.

## Установка и запуск

Требования:
- Python 3.10+
- Tkinter
- Pillow (для сохранения PNG)

Установка зависимостей:

```bash
pip install pillow
```

Запуск:

```bash
python main.py
```

## Структура проекта

- `main.py` — точка входа GUI.
- `ui.py` — основная логика интерфейса и сценарии.
- `ui_content.py` — статические тексты и встроенные примеры.
- `ui_tree_canvas.py` — рендер/очистка дерева на Canvas.
- `grammar.py` — публичный API парсинга грамматики.
- `grammar_lexer.py` — низкоуровневый лексический разбор RHS.
- `cs_representation.py` — построение и форматирование CS-представления.
- `parse_tree.py` — построение дерева разбора.
- `test/` — набор black-box тестов.
